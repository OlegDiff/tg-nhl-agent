# Release scope v1 (DoD)

## Цель v1
Ежедневно в **08:00 MSK** бот публикует **одно сообщение** в Telegram-канал с перечнем матчей NHL за последние **48 часов (отсчёт от 08:00 MSK)**, отсортированных по индексу интересности, с **двумя ссылками** на каждый матч: highlights и full game (если найдены). Без спойлеров.

## Что входит в v1 (Must-have)
1) **Один запуск = один пост** (без дублей).
2) **Окно матчей:** [08:00 MSK − 48h, 08:00 MSK).
3) **Скоринг/ранжирование:** по таблице “повышающих очков” (игроки/команды) + базовые факторы (допускается использование счёта/стат для расчёта).
4) **Порог интересности:** публикуются только матчи с индексом ≥ `MIN_INTEREST` (значение в конфиге).
5) **Видео:** на каждый матч ищется:
   - 1 ссылка highlights
   - 1 ссылка full game
   Если конкретная ссылка не найдена — в посте рядом с матчем пишется “ссылка не найдена” (по типу ролика).
6) **Ошибки/частичные данные:** если часть интеграций недоступна, пост всё равно публикуется с тем, что есть, и секцией “⚠️ Проблемы” внизу.
7) **Без спойлеров:** в тексте поста **нет счёта, исхода, подробных игровых событий**.

## Выходные форматы (v1)
- **Telegram message text**:
  - блоки по матчам: “TEAM1 — TEAM2”, затем строки `highlights:` и `full:`
  - внизу опционально “⚠️ Проблемы: …”

## Что НЕ входит в v1 (Explicit non-goals)
- несколько постов в день, ручные команды управления ботом
- персонализация под нескольких пользователей
- сложные объяснения “почему матч интересен”
- красивое форматирование (эмодзи/карточки/кнопки) кроме базовой читаемости
- автоповтор публикации при сомнительном статусе (только защита от дублей)

## Критерии приёмки (Acceptance)
- За 1 запуск в 08:00 MSK появляется **ровно 1** пост в канале.
- При повторном запуске в тот же день **дубль не появляется**.
- В посте нет спойлеров.
- Если видео не найдено — это явно отражено на уровне конкретного матча/типа ролика.
- При недоступности одной из интеграций пост содержит “⚠️ Проблемы”.
